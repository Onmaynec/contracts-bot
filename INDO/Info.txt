Я разрабатываю Discord-бота на Node.js (ES Modules) с использованием discord.js v14 и SQLite.

## 📦 Технологии

* Node.js v24+
* discord.js v14
* SQLite (sqlite3)
* Архитектура:

  * commands/
  * events/
  * services/
  * database/

---

## 🎯 Основная задача бота

Бот управляет таблицей активности контрактников (ежедневной).

Каждый день создаётся новая таблица (embed сообщение), где пользователи отмечаются.

---

## 📊 Формат таблицы

Заголовок:
📋 Онлайн контрактов на ДД.ММ.ГГГГ

Разделы:
🟢 Онлайн:
**Nick | Время**

🟡 Форс-мажор:
Nick | Время | `Причина`

🔴 Оффлайн:
*Nick | Время*

---

## 👤 Логика пользователя

* Один пользователь = одна запись в таблице (уникально по user_id + guild_id)
* Имя берётся как displayName (серверный ник)

---

## 🔘 Кнопки под таблицей

1. 🟢 Онлайн
   → открывает модалку
   → пользователь вводит прайм-тайм (например 12:00-22:00)
   → создаётся/обновляется запись
   → status = online
   → сохраняется time

2. 🟡 Форс-мажор
   → открывает модалку
   → ввод причины
   → status = afk
   → сохраняется reason
   → time НЕ меняется

3. 🔴 Оффлайн
   → сразу меняет статус на offline
   → time НЕ меняется

4. 🔵 Прайм-тайм
   → модалка
   → обновляет только поле time

---

## 🧠 ВАЖНОЕ ПОВЕДЕНИЕ

* Время задаётся при первом нажатии "Онлайн"
* При смене статусов (форс/оффлайн) время сохраняется
* При повторном "Онлайн" время можно обновить
* Форс-мажор перезаписывает причину
* Дубли записей запрещены (через UNIQUE INDEX)

---

## 🗄️ Структура БД

Таблица users:

* id (PK)
* user_id (TEXT)
* guild_id (TEXT)
* name (TEXT)
* status (TEXT: online / afk / offline)
* time (TEXT)
* reason (TEXT)
* date (TEXT)

Индекс:
UNIQUE(user_id, guild_id)

---

## ⚙️ Основные файлы

* interactionCreate.js → вся логика кнопок и модалок
* tableService.js → генерация embed таблицы
* db.js → подключение и инициализация SQLite

---

## 📌 Текущее состояние

* Таблица работает
* Кнопки работают
* Модалки работают
* Данные сохраняются
* Нет дублей
* Обновление сообщения происходит через message.edit()

---

## ❗ Что важно учитывать при доработке

* Не ломать текущую логику статусов
* Не создавать дубликаты пользователей
* Всегда использовать одно сообщение (редактирование)
* Соблюдать формат таблицы (жирный / курсив / код)
* Использовать displayName вместо username
